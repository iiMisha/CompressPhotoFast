# План: Исправление повреждения изображений при высоком CPU

## Диагноз (уточнённый)

**Симптомы:** Частичное повреждение (верхняя часть видна, нижняя серая/отсутствует) в **режиме замены**.

Частичный JPEG с видимым верхом и серым низом = **файл был обрезан/усечён во время записи**. Анализ кода выявил точную причину.

### Корневая причина: ExifInterface.saveAttributes() перезаписывает файл (КРИТИЧЕСКАЯ)

**Цепочка событий в режиме замены:**

1. `MediaStoreUtil.saveCompressedImageFromStream()` → `safeUpdateExistingFile()` — корректно записывает сжатые данные через `"wt"` (truncate + write) ✅
2. `clearIsPendingFlag()` — снимает флаг IS_PENDING ✅
3. **`ExifUtil.handleExifForSavedImage()` → `applyExifFromMemory()` → `exif.saveAttributes()`** ❌

`ExifInterface.saveAttributes()` (строка 852 в `ExifUtil.kt`) **перезаписывает файл целиком**:
- Читает весь JPEG из файлового дескриптора
- Модифицирует EXIF-сегмент
- **Записывает файл заново поверх старого**

При высоком CPU/памяти этот процесс может:
- Упасть посередине записи → файл обрезан → верхняя часть видна, нижняя потеряна
- Быть прерван системой (OOM kill, ANR kill) → неполная запись
- Прочитать повреждённые данные (если bitmap.compress() выдал неполный JPEG под OOM)

**Дополнительный фактор:** В режиме замены оригинал **удаляется** после сохранения. Если `saveAttributes()` повреждает файл, пользователь теряет и оригинал, и получает битый файл.

### Сопутствующие проблемы:

1. **Нет верификации JPEG после compress()** — повреждённые данные не обнаруживаются
2. **Нет верификации после saveAttributes()** — повреждение не откатывается
3. **Параллельность WorkManager** — до 4 воркеров одновременно увеличивают нагрузку
4. **Нет backup перед EXIF-записью** — невозможно восстановить файл

## План исправлений

### Шаг 1: Атомарная запись EXIF с backup (ГЛАВНЫЙ FIX)
**Файл:** `ExifUtil.kt` — метод `applyExifFromMemory()`

**Суть:** Перед `saveAttributes()` создать backup копию файла. При ошибке — восстановить из backup.

```kotlin
// В applyExifFromMemory(), перед exif.saveAttributes():

// 1. Создаём backup копию файла
val backupFile = File(context.cacheDir, "exif_backup_${System.currentTimeMillis()}.jpg")
context.contentResolver.openInputStream(uri)?.use { input ->
    FileOutputStream(backupFile).use { output ->
        input.copyTo(output)
    }
}

try {
    exif.saveAttributes()
    // Успех — удаляем backup
    backupFile.delete()
} catch (e: Exception) {
    // Ошибка — восстанавливаем из backup
    LogUtil.error(uri, "EXIF backup", "saveAttributes() упал, восстанавливаем файл из backup")
    try {
        context.contentResolver.openOutputStream(uri, "wt")?.use { output ->
            backupFile.inputStream().use { input ->
                input.copyTo(output)
            }
        }
    } catch (restoreError: Exception) {
        LogUtil.error(uri, "EXIF restore", "Критическая ошибка: не удалось восстановить файл из backup", restoreError)
    }
    throw e
} finally {
    backupFile.delete()
}
```

### Шаг 2: Верификация JPEG после compress()
**Файл:** `ImageCompressionUtil.kt` — метод `compressImageToStream()`

После `inputBitmap.compress()` (строка 380) добавить валидацию:

```kotlin
val success = inputBitmap.compress(Bitmap.CompressFormat.JPEG, quality, outputStream)
if (!success) { ... }

// Валидация: проверяем целостность JPEG данных
val compressedBytes = outputStream.toByteArray()
val validationOptions = BitmapFactory.Options().apply { inJustDecodeBounds = true }
val validationBitmap = BitmapFactory.decodeByteArray(compressedBytes, 0, compressedBytes.size, validationOptions)
if (validationBitmap == null || validationBitmap.outWidth <= 0 || validationBitmap.outHeight <= 0) {
    LogUtil.error(uri, "Сжатие", "ВАЛИДАЦИЯ НЕ ПРОШЛА: JPEG данные повреждены (decodeByteArray вернул null)")
    return@withContext null
}
LogUtil.debug("Сжатие", "Валидация JPEG прошла: ${validationBitmap.outWidth}x${validationBitmap.outHeight}")

// Пересоздаём outputStream из валидированных данных
outputStream.reset()
outputStream.write(compressedBytes)
```

**Примечание:** `toByteArray()` + `decodeByteArray()` добавляет ~50-100мс, но гарантирует целостность. Альтернатива — использовать `ByteArrayOutputStream.toByteArray()` только для валидации, а потом передать `toInputStream()` как раньше.

### Шаг 3: Верификация сохранённого файла в Worker перед удалением оригинала
**Файл:** `ImageCompressionWorker.kt`

После сохранения (строка ~302), перед удалением оригинала (строка ~327) добавить:

```kotlin
// Верификация сохранённого файла перед удалением оригинала
if (FileOperationsUtil.isSaveModeReplace(appContext)) {
    val isSavedFileValid = verifySavedImageIntegrity(appContext, savedUri)
    if (!isSavedFileValid) {
        LogUtil.error(imageUri, "Верификация", 
            "КРИТИЧЕСКАЯ ОШИБКА: Сохранённый файл повреждён! Отмена удаления оригинала")
        // НЕ удаляем оригинал, возвращаем failure
        StatsTracker.updateStatus(imageUri, StatsTracker.COMPRESSION_STATUS_FAILED)
        return@withContext Result.failure()
    }
}

// Новый приватный метод:
private suspend fun verifySavedImageIntegrity(context: Context, uri: Uri): Boolean = withContext(Dispatchers.IO) {
    try {
        val options = BitmapFactory.Options().apply { inJustDecodeBounds = true }
        context.contentResolver.openInputStream(uri)?.use { inputStream ->
            val bitmap = BitmapFactory.decodeStream(inputStream, null, options)
            if (bitmap == null || options.outWidth <= 0 || options.outHeight <= 0) {
                LogUtil.error(uri, "Верификация", "Файл не является корректным изображением")
                return@withContext false
            }
        } ?: return@withContext false
        return@withContext true
    } catch (e: Exception) {
        LogUtil.error(uri, "Верификация", "Ошибка при проверке целостности файла", e)
        return@withContext false
    }
}
```

**Файл:** `ExifUtil.kt` — метод `handleExifForSavedImage()`

После `applyExifFromMemory()` добавить верификацию что файл не повреждён:

```kotlin
if (exifSuccess) {
    // Верификация: файл должен быть читаемым после saveAttributes()

        LogUtil.error(destinationUri, "EXIF", "Файл повреждён после saveAttributes()")
```
