# План: Исправление IllegalStateException (pending items) + избыточное логирование

## Проблема 1 (КРИТИЧЕСКАЯ): `IllegalStateException: Only owner is able to interact with pending item`

### Описание ошибки

При съёмке фото камерой приложение пытается прочитать EXIF-данные файла, который ещё в состоянии `is_pending=1` (камера ещё пишет файл). Это вызывает `IllegalStateException` с полным стектрейсом в логе.

### Цепочка вызовов

```
MediaStoreObserver.contentObserver.onChange()   (MediaStoreObserver.kt:109)
  → delay(CONTENT_OBSERVER_DELAY_SECONDS)
  → ExifUtil.getCompressionMarker()              (ExifUtil.kt:1108)
    → ExifUtil.getExifInterface()                 (ExifUtil.kt:299)
      → MediaStore.setRequireOriginal(uri)        (создаёт ?requireOriginal=1 URI)
      → openInputStream(finalUri)                 💥 CRASH: IllegalStateException
```

Вторая ошибка в цепи:
```
MediaStoreObserver.processUriWithRetry()          (MediaStoreObserver.kt:140)
  → UriUtil.isFilePending()                       (UriUtil.kt:503)
    → openInputStream(uri)                        💥 CRASH: IllegalStateException
    → LogUtil.warning("Ошибка при проверке is_pending...")  (UriUtil.kt:511)
```

### Корневые причины

1. **`ExifUtil.getExifInterface()`** (стр.298-305): ловит только `FileNotFoundException`, но не `IllegalStateException`
2. **`MediaStoreObserver`** (стр.108-109): проверяет EXIF ДО проверки `is_pending` — нужно наоборот
3. **`UriUtil.isFilePending()`** (стр.503): для `is_pending=1` файлов с `size=0` вызывает `openInputStream()`, что тоже падает с `IllegalStateException`
4. **`MediaStore.setRequireOriginal()`** (стр.287): добавляет `?requireOriginal=1` к URI, что усиливает проверку владельца для pending-файлов

## План исправлений

### Шаг 1 (КРИТИЧЕСКИЙ): Исправить `ExifUtil.getExifInterface()` — обработка `IllegalStateException`

**Файл:** `app/src/main/java/com/compressphotofast/util/ExifUtil.kt`

В методе `getExifInterface()` (стр.281-310) добавить обработку `IllegalStateException`:

```kotlin
// ТЕКУЩИЙ КОД (стр.298-305):
try {
    context.contentResolver.openInputStream(finalUri)?.use { inputStream ->
        return ExifInterface(inputStream)
    }
} catch (e: FileNotFoundException) {
    LogUtil.error(uri, "Получение EXIF", "Файл не найден: ${e.message}")
    return null
}

// ИСПРАВЛЕННЫЙ КОД:
try {
    context.contentResolver.openInputStream(finalUri)?.use { inputStream ->
        return ExifInterface(inputStream)
    }
} catch (e: FileNotFoundException) {
    LogUtil.error(uri, "Получение EXIF", "Файл не найден: ${e.message}")
    return null
} catch (e: IllegalStateException) {
    // Файл в состоянии is_pending — другой приложение ещё пишет его
    // "Only owner is able to interact with pending item"
    LogUtil.processDebug("EXIF: файл ещё в процессе записи (pending), пропускаем: $uri")
    return null
}
```

Также в блоке `setRequireOriginal()` (стр.284-296) добавить fallback при `IllegalStateException`:

```kotlin
val finalUri = try {
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q &&
        uri.toString().startsWith("content://media/")) {
        MediaStore.setRequireOriginal(uri)
    } else {
        uri
    }
} catch (e: IllegalStateException) {
    // setRequireOriginal может бросить исключение для pending-файлов
    LogUtil.processDebug("EXIF: setRequireOriginal failed, используем обычный URI: $uri")
    uri
} catch (e: Exception) {
    LogUtil.processWarning("Ошибка при получении оригинального URI для EXIF: ${e.message}")
    uri
}
```

### Шаг 2 (КРИТИЧЕСКИЙ): Проверка `is_pending` ДО чтения EXIF в MediaStoreObserver

**Файл:** `app/src/main/java/com/compressphotofast/util/MediaStoreObserver.kt`

В `contentObserver.onChange()` (стр.106-116) добавить проверку pending ПЕРЕД EXIF:

```kotlin
// ТЕКУЩИЙ КОД (стр.106-116):
val delayJob = handlerScope.launch {
    delay(Constants.CONTENT_OBSERVER_DELAY_SECONDS * 1000L)
    val (isAlreadyCompressed, _, compressionTimestamp) = withContext(Dispatchers.IO) {
        ExifUtil.getCompressionMarker(context, it)  // CRASH для pending файлов
    }
    ...
    processUriWithRetry(it, uriString)
}

// ИСПРАВЛЕННЫЙ КОД:
val delayJob = handlerScope.launch {
    delay(Constants.CONTENT_OBSERVER_DELAY_SECONDS * 1000L)
    
    // СНАЧАЛА проверяем is_pending — ПЕРЕД чтением EXIF
    if (UriUtil.isFilePending(context, it)) {
        LogUtil.processDebug("MediaStoreObserver: файл ещё в процессе записи (pending), планируем повтор: $it")
        processUriWithRetry(it, uriString)
        return@launch
    }
    
    // Только после проверки pending — читаем EXIF
    val (isAlreadyCompressed, _, compressionTimestamp) = withContext(Dispatchers.IO) {
        ExifUtil.getCompressionMarker(context, it)
    }
    if (isAlreadyCompressed && (System.currentTimeMillis() - compressionTimestamp) < 60_000L) {
        LogUtil.processDebug("MediaStoreObserver: URI $it имеет свежий маркер сжатия (< 60 сек), пропускаем")
        pendingTasks.remove(uriString)
        return@launch
    }
    processUriWithRetry(it, uriString)
}
```

### Шаг 3 (КРИТИЧЕСКИЙ): Исправить `UriUtil.isFilePending()` — не вызывать `openInputStream()` для pending-файлов

**Файл:** `app/src/main/java/com/compressphotofast/util/UriUtil.kt`

В методе `isFilePending()` (стр.499-513) обработать `IllegalStateException`:

```kotlin
// ТЕКУЩИЙ КОД (стр.501-513):
if (sizeIsNull || size == 0L) {
    try {
        context.contentResolver.openInputStream(uri)?.use { stream ->
            if (stream.available() > 0) {
                LogUtil.processDebug("Файл имеет is_pending=1 и _size=..., но физически существует: $uri")
                return false
            }
        }
    } catch (e: Exception) {
        // Проблема: логирует IllegalStateException как WARNING
        LogUtil.warning(uri, "isPending", "Ошибка при проверке is_pending, считаем файл pending: ${e.message}")
        return true
    }
}

// ИСПРАВЛЕННЫЙ КОД:
if (sizeIsNull || size == 0L) {
    try {
        context.contentResolver.openInputStream(uri)?.use { stream ->
            if (stream.available() > 0) {
                LogUtil.processDebug("Файл имеет is_pending=1 и _size=..., но физически существует: $uri")
                return false
            }
        }
    } catch (e: IllegalStateException) {
        // "Only owner is able to interact with pending item" — файл точно pending
        return true
    } catch (e: Exception) {
        LogUtil.processDebug("isPending: файл недоступен, считаем pending: ${e.message}")
        return true
    }
}
```
