# Правила разработки CompressPhotoFast

## Настройки
- **Язык общения**: Русский.
- **ОС**: Linux Mint.

## Сборка
ОБЯЗАТЕЛЬНО: После КАЖДОГО изменения кода выполняйте сборку измененного модуля (или полного проекта при изменениях в нескольких) для проверки. При составлении планов разработки также НЕОБХОДИМО указывать о необходимости производить сборку и исправлять ошибки после каждого шага (см. раздел "Планирование разработки").

## Тестирование
ОБЯЗАТЕЛЬНО: При изменении кода необходимо создавать тесты и проводить тестирование.

### Правила написания тестов
- Для каждого нового класса или функции создаются соответствующие unit тесты
- При изменении существующего кода обновляются или добавляются тесты для проверки изменений
- Тесты должны покрывать основные сценарии использования и граничные случаи
- Используйте базовые классы тестирования: `BaseUnitTest` для unit тестов и `BaseInstrumentedTest` для instrumentation тестов

### Запуск тестов

**Только unit тесты:**
```bash
./gradlew testDebugUnitTest
```

**Все тесты (unit + instrumentation):**
```bash
./scripts/run_all_tests.sh
```

**Проверка покрытия кода:**
```bash
./gradlew jacocoTestReport
```
HTML отчет покрытия: `app/build/reports/jacoco/jacocoTestReport/html/index.html`

### Полная сборка
```bash
./gradlew assembleDebug
```
